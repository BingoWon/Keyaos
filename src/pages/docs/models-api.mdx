# Models API

Browse the complete model catalog with pricing, capabilities, and provider availability.

## GET /v1/models

Returns all available models. Each model appears exactly once — pricing reflects **real transaction prices** updated every minute.

**Request:**

```bash
curl https://your-gateway.workers.dev/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**


## Response Fields

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Model identifier, used in `/v1/chat/completions` requests |
| `name` | `string` | Human-readable display name |
| `created` | `number` | Unix timestamp (seconds) when the model was first available |
| `description` | `string \| null` | Detailed model description and capabilities |
| `hugging_face_id` | `string \| null` | HuggingFace model card reference |
| `context_length` | `number \| null` | Maximum context window in tokens |
| `pricing` | `object` | Cost in **USD per token** (string) — see below |
| `architecture` | `object \| null` | Model architecture and modality info |
| `supported_parameters` | `string[] \| null` | API parameters this model accepts |
| `providers` | `string[]` | Upstream providers that can serve this model |

## Pricing Object

All values are strings representing **USD per token**. `prompt` and `completion` are updated every minute based on actual transaction data. Other dimensions are derived proportionally.

| Field | Description |
|---|---|
| `prompt` | Input text token cost |
| `completion` | Output text token cost |
| `image` | Image input token cost |
| `audio` | Audio input token cost |
| `internal_reasoning` | Thinking/reasoning token cost |
| `input_cache_read` | Cached prompt read cost |
| `input_cache_write` | Cache write cost |

## Architecture Object

| Field | Type | Description |
|---|---|---|
| `modality` | `string` | Summary string, e.g. `"text+image->text"` |
| `input_modalities` | `string[]` | Accepted input types |
| `output_modalities` | `string[]` | Produced output types |
| `tokenizer` | `string` | Tokenizer family, e.g. `"Gemini"`, `"Claude"` |
| `instruct_type` | `string \| null` | Instruction format, e.g. `"deepseek-r1"` |
